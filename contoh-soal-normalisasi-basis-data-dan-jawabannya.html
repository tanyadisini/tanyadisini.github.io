1. kenapa harus ada normalisasi dalam basis data ?


Tujuan Normalisasi Database

Tujuan normalisasi database adalah untuk menghilangkan dan mengurangi redudansi data dan tujuan yang kedua adalah memastikan dependensi data (Data berada pada tabel yang tepat).
Jika data dalam database tersebut belum di normalisasi maka akan terjadi 3 kemungkinan yang akan merugikan sistem secara keseluruhan.

INSERT Anomali : Situasi dimana tidak memungkinkan memasukkan beberapa jenis data secara langsung di database.
DELETE Anomali: Penghapusan data yang tidak sesuai dengan yang diharapkan, artinya data yang harusnya tidak terhapus mungkin ikut terhapus.
UPDATE Anomali: Situasi dimana nilai yang diubah menyebabkan inkonsistensi database, dalam artian data yang diubah tidak sesuai dengan yang diperintahkan atau yang diinginkan.

2. kenapa normalisasi data base begitu penting dalam basis data​


Jawaban:

untuk menghilangkan dan mengurangi redudansi data dan tujuan yang kedua adalah memastikan dependensi data (Data berada pada tabel yang tepat).

Jawaban:

Normalisasi database sangat penting dalam basis data karena dapat meningkatkan kualitas dan efisiensi penyimpanan data. Normalisasi bertujuan untuk mengorganisir data dalam tabel yang memiliki struktur yang terdefinisi dengan baik, sehingga mengurangi duplikasi data dan meningkatkan konsistensi data.

Dengan normalisasi, setiap data hanya disimpan dalam satu tempat saja, yaitu dalam tabel yang sesuai. Hal ini dapat mengurangi redundansi dan anomali data yang mungkin terjadi pada struktur database yang tidak ter-normalisasi. Contohnya, jika sebuah database memiliki beberapa tabel dengan informasi yang serupa dan ada duplikasi data, maka akan memakan ruang penyimpanan yang lebih besar dan sulit dalam pemeliharaan data. Normalisasi dapat mengurangi duplikasi data tersebut dan membantu dalam menjaga integritas data.

Selain itu, normalisasi juga dapat meningkatkan efisiensi dalam melakukan operasi basis data seperti pencarian, penyimpanan, dan pembaruan data. Normalisasi database dapat memudahkan dalam mengakses data dan mempercepat waktu pencarian data yang dibutuhkan, sehingga mempercepat proses pengambilan keputusan.

Dalam rangka memaksimalkan kinerja basis data dan menghindari anomali data, normalisasi database adalah langkah yang penting dan sangat dianjurkan untuk dilakukan pada setiap tahap pengembangan database.


3. apa tujuan dan proses dari normalisasi basis data?


Fungsi Normalisasi Database Pada ilmu database atau basis data, normalisasi digunakan untuk menghindari terjadinya berbagai anomali data dan tidak konsistensinya data. Ini merupakan funsi secara umum. Dalam beberapa kasus normalisasi ini sangat penting untuk menunjang kinerja database dan memastikan bahwa data dalam database tersebut aman dan tidak terjadi kesalahan jika mendapat perintah SQL terutama DML yaitu update, insert, dan delete. Perlu diketahui dalam beberapa kasus Normalisasi database terkadang harus diubah menjadi bentuk denormalisasi, terutama untuk data yang telah besar dan membengkak. Denormalisasi ini ditujukan untuk meningkatkan performance dengan meletakkan beberapa field menjadi satu tabel sehingga mudah di tarik. Denormalisasi ini sering digunakan untuk menarik data yang besar dari database.

14. Apa hubungan Normalisasi dengan Redundansi Data dan Konsistensi Data?


Jawaban:

Normalisasi Data adalah proses evaluasi dan perbaikan struktur tabel untuk meminimalisir

(mengurangi) kemungkinan adanya anomali data.

Tahapannya ialah :

* Tiap tabel mewakili subjek tunggal

* Item data hanya disimpan dan diupdate dalam 1 tabel

* Semua atribut yang bukan PK bergantung hanya pada PK

* Tiap tabel terbebas dari anomali insert , delete atau update

* Redundansi Data ialah duplikasi data dalam suatu file sehingga menyebabkan terjadinya pemborosan tempat penyimpanan.

* Konsistensi Data ialah data yang bersifat tetap

Data yang tersimpan dalam lokasi berbeda tidak terupdate secara konsisten sehingga menyebabkan redundansi data .

Redundansi yang tidak terkendali menyebabkan Keamanan data yang buruk , Anomali Data , Ketidakkonsistensi data .

Penjelasan:

u Model REA adalah suatu alat pemodelan konseptual yang khusus dirancang untuk

melengkapi struktur dalam perancangan database SIA. Dalam model REA ditentukan yaitu entity apa yang harus disertakan dalam database SIA dan bagaimana susunan relationship antara entity dalam database SIA.Tipe entity dalam model REA dibedakan dalam tiga kategori, yaitu:

u Resources,Events, dan Agents. 

1. Resources didefinisikan sebagai sesuatu yang memiliki nilai

ekonomis bagi organisasi tersebut. Contoh resources adalah kas, inventaris, peralatan,

persediaan, gudang, pabrik, dan tanah. 

2. Events menunjukkan aktivitas-aktivitas bisnis,

dimana manajemen ingin mengumpulkan informasi untuk tujuan perencanaan atau

pengawasan. Sebagai contoh, aktivitas penjualan akan mengurangi persediaan dan

aktivitas penerimaan kas akan menambah jumlah kas. SIA harus dirancang untuk

memperoleh dan menyimpan informasi aktivitas tersebut. Sedangkan 

3. Agents adalah

orang dan organisasi yang berpartisipasi dalam aktivitas dan kepada siapa informasi

diserahkan untuk tujuan perencanaan, pengawasan, dan pengevaluasian. Contoh agent adalah pengawai, pelanggan, dan pemasok. Berikut contoh kasus dibawah ini:

34. contoh control file pada administrasi basis data ?


View-view dictionary yang datanya diambil dari control file di antaranya adalah: v$DATABASE, v$DATAFILE, v$TEMPFILE, v$LOGFILE, V$LOG, dan lain-lain. Control file dibaca Oracle ketika instance sedang mount. Oleh karena itu, view-view tersebut bisa di-query meskipun database dalam keadaan mount (belum open).

Lokasi 
Lokasi control file ditunjukkan oleh parameter control_files. Silahkan temukan parameter control_files ini di instance parameter (init file atau spfile). Atau gunakan SQL command berikut:

SQL> show parameter control_files SQL> -- atau SQL> select name from v$controlfile; SQL> -- atau SQL> select NAME,VALUE from v$parameter where NAME ='control_files';

Perbedaan Basis Data dengan Sistem Basis Data: Basis Data adalah kumpulan data yang disimpan dan dikelola secara terstruktur, sedangkan Sistem Basis Data adalah perangkat lunak yang digunakan untuk mengakses, mengelola, dan memanipulasi basis data.Basis Data Relasional: Model basis data yang menggunakan tabel untuk menyimpan data, dan hubungan antara tabel dijaga melalui kunci asing.Normalisasi dalam basis data: Proses pengorganisasian data dalam basis data untuk mengurangi redudansi dan meningkatkan integritas data. Tingkatan normalisasi meliputi 1NF, 2NF, 3NF, dan seterusnya.Primary Key, Foreign Key, dan Unique Key: Primary Key adalah kolom atau kombinasi kolom yang secara unik mengidentifikasi setiap baris dalam tabel. Foreign Key adalah kolom yang mengacu pada Primary Key di tabel lain, dan digunakan untuk menjaga hubungan antar-tabel. Unique Key adalah kolom atau kombinasi kolom yang memiliki nilai unik untuk setiap baris, namun boleh memiliki nilai NULL.Aplikasi komputer dalam pembuatan sistem basis data: MySQL, Oracle, SQL Server, PostgreSQL, MongoDB.Diagram dalam pemodelan data dalam basis data: Entity-Relationship (ER) diagram, Relational diagram, Schema diagram, Normalization diagram.Contoh basis data dengan 3 tabel: Tabel "Produk" dengan kolom seperti "ID Produk", "Nama Produk", "Harga", Tabel "Pelanggan" dengan kolom seperti "ID Pelanggan", "Nama", "Alamat", dan Tabel "Transaksi" dengan kolom seperti "ID Transaksi", "ID Pelanggan", "ID Produk", "Jumlah".
